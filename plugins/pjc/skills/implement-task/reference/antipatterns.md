# Antipatterns — implement-task

자율 루프에서 절대 금지하는 패턴 목록.

## 검증·완료 관련

| 안티패턴 | 올바른 행동 |
|---|---|
| "구현이 맞아 보입니다"로 완료 선언 | V-1~V-6 모두 통과 후 증거 첨부 |
| Review subagent 생략 | 의무 — 절대 생략 금지 |
| Review 결과를 자체 판단으로 묵살 | 이슈 0까지 반복 또는 Halt |
| 모든 변경에 전체 코드베이스 리뷰 | 해당 task diff로 범위 한정 |
| 동일 오류를 다른 방식으로 5회 시도 | Halt — 사용자 보고 |
| 사소한 빌드 오류로 사용자 호출 | 자체 수정 |
| **"잘 동작할 것이다"로 검증 생략** | **빌드 + 테스트 통과 + 2단계 리뷰 통과 후에만 완료** |

## 자율 루프 위반

| 안티패턴 | 올바른 행동 |
|---|---|
| "이건 사용자가 결정해야..." | plan.md 결함 — Halt → plan-feature 복귀 |
| **task 완료 후 "다음 진행할까요?"라고 묻기** | **즉시 다음 task의 Phase P 시작 (사용자 응답 기다리지 않음)** |
| **"T2 시작하면 될까요?" 같은 확인 요청** | **plan.md = 전체 위임장. 이미 승인받음. 그냥 진행** |
| **MINOR follow-up 등록 시 사용자 의견 묻기** | **plan.md/follow-up 섹션에 기록 후 계속 진행** |
| **"이 부분 어떻게 처리할까요?" 사소한 결정에 사용자 호출** | **plan-feature Decision Points에서 결정되었어야 함. 결정 누락이면 Halt** |

## 범위·일관성 위반

| 안티패턴 | 올바른 행동 |
|---|---|
| 무관한 리팩토링 끼워넣기 | plan.md에 새 task 등록 |
| 사용처 1곳인데 헬퍼 추출 | 인라인 유지 |
| 1500라인 초과 파일에 계속 추가 | 분리 작업을 plan에 등록 |
| 영문 주석 자동 생성 | 한글 주석 |
| 환각 메서드 호출 | 문서·소스 확인 또는 Halt |

## Cross-File 위반 (a 수정 → b, c 누락)

| 안티패턴 | 올바른 행동 |
|---|---|
| **A 파일만 수정하고 호출자 B, C 그대로 두기** | **Phase P-3에서 grep으로 호출자 추적 → 함께 수정** |
| **메서드 시그니처를 추측해서 호출** | **Read로 정의 확인 후 호출** |
| **새 메서드 추가 후 호출자 갱신 없이 완료** | **모든 사용처를 같은 task에서 갱신** |
| **인터페이스 추가/변경 후 구현체 일부만 갱신** | **모든 구현체를 같은 task에서 갱신** |
